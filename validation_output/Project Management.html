7. Explicit Risk Notes

The following risk notes should be reviewed by the proposal team:

Unanswered Requirement (Row 24)

Row 24 ('Create a procurement plan against a project') has no availability code and only a 'Not Clear' comment. This is the only unanswered item in the tab and represents a significant gap. D365 F&O natively supports project procurement planning — this should be marked 'S' with a clear explanation. Leaving it unanswered may cause the evaluator to assume the system does not support this capability, which would be incorrect and potentially disqualifying for this section.

Modification Classifications May Be Overstated (Rows 21, 27)

Two items are marked 'M' (Modification) where D365 F&O built-in Alert Rules and Workflow Notifications could address the requirements with configuration only. Overstating the need for modifications increases perceived implementation cost and effort, which may negatively affect scoring. These should be reviewed for potential reclassification to 'S' or 'W'.

MS Project Integration Dependency (Row 35)

The MS Project integration response is correct but should explicitly mention the dependency on the Microsoft Project Client Add-in. Without this qualifier, the client may assume native browser-based MS Project integration, which is not how D365 F&O delivers this capability. Failure to set proper expectations could lead to post-implementation disputes.

Assessment Sheets Capability (Row 37)

Marking 'Project Assessment Sheets' as 'S' may overstate the out-of-the-box capability. D365 F&O does not have pre-built, structured multi-departmental assessment templates with KB/lessons learned. While the building blocks exist (Questionnaires, Document Management, Notes), assembly of a complete assessment sheet solution requires configuration effort. This should be disclosed to avoid scope disputes.

Travel Timesheet Specifics (Row 39)

While D365 F&O timesheets and expense management cover the core requirements, specific fields for arrival/departure times are not standard timesheet line fields. Minor form customization may be needed. The 'S' marking is defensible but should include a qualifying note.
