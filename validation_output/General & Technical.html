Explicit Risk Notes

HIGH   RISK-GT-001 — ODBC Access Claim May Mislead Client (R15)

The response claims 'S' (Standard) for ODBC access, but D365 F&O Cloud does not expose the database via ODBC. If the client's existing tools rely on direct ODBC connections to the ERP database, this will fail during implementation. The BYOD approach requires setting up an Azure SQL Database, configuring data export jobs, and managing data freshness — a fundamentally different architecture from direct ODBC.

Impact: Implementation delay, potential scope change, client dissatisfaction if expectations are set for direct database connectivity.

Mitigation: Clarify during pre-sales that data access is via OData APIs and BYOD. Demonstrate the Data Entity framework and BYOD export during a technical workshop.

HIGH   RISK-GT-002 — Entire Remote User Policy Section Unanswered (R51-R55)

Five requirements under 'Remote User Policy' have no availability codes. This is a significant gap in a compliance matrix. From a D365 F&O perspective, remote vs. local user policies are a distinction that does not apply — Azure AD governs authentication uniformly.

Impact: The evaluator may score these as 'not met', resulting in point deductions. It also signals incomplete vendor due diligence.

Mitigation: Populate R52-R55 with the same codes as R47-R50 and add a comment explaining that Azure AD provides uniform security policies regardless of user location.
